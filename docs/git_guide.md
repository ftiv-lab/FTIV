
---

## 1. 「作業中の変更」をなかったことにしたい時
「コードをいじってみたけど、やっぱり全部やめたい！さっきのセーブポイントに戻して！」という時。

```bash
# 全ファイルの変更を破棄して、前回のコミット状態に戻す（危険！）
git reset --hard HEAD
```

*   **注意**: コミットしていない編集内容はすべて消えます。取り消せません。

## 2. 過去のバージョンを「ちょっと見たい」時
「昨日の時点ではどう書いてたっけ？」を確認したい時。タイムトラベル。

```bash
# 1. 履歴のID（ハッシュ値）を確認
git log --oneline
# 表示例: a1b2c3d Initial commit ...

# 2. その過去の世界に移動
git checkout a1b2c3d
```

*   **戻り方**: `git checkout master` で現代に戻れます。

## 3. 分岐（ブランチ）の活用
「今のマインドマップ機能を温存したまま、機能を削除したシンプル版を作りたい」場合。

1.  今の状態を `feature/mindmap_v1` という名前で保存しておく（すでに作成済み）。
2.  `master` ブランチに戻る (`git checkout master`)。
3.  `master` ブランチでファイルを削除したり修正したりする。
4.  いつでも `git checkout feature/mindmap_v1` でマインドマップ版に戻れる。
